## データがどこにあるか（これを間違えると迷子になる）

| データ | 置き場所 | 書き込み方法 |
|---|---|---|
| 予定・買うもの（☁️同期） | **このリポジトリ直下の `yotei.json`**（スプシではない！） | GitHubにコミット（下記フォーマット） |
| 買うものメモ（端末） | 各端末のlocalStorage `nishimura_shop_v1` | 西村OSの🛒タブから手入力のみ。AIは書けない |
| 欲しいものリスト（大きい買い物） | スプシ「西村家ファイナンスDB_v2」の「欲しいものリスト」シート | 従来どおり（行を渡して西村が貼る等） |
| 家計・資産・健康 | 同スプシの各シート（CSV公開→⚙設定のURL経由で表示） | **読み取り専用。AIは書き込まない** |
| 勉強（宅建アプリ） | `study/` 配下（単一HTML、データ内蔵） | MacのClaude Codeがビルドしてコミット |

### ⚠️ よくある誤解（過去に実際に起きた）
- ❌ 「買うもの・予定はスプシの『買うもの』タブと同期している」→ **そんなタブは存在しない**。同期の実体は `yotei.json`
- ❌ 予定を「欲しいものリスト」シートに入れる → 欲しいものリストは大きい買い物の計画専用。**日々の予定・買い物は `yotei.json`**
- ❌ パスワードや認証情報を置く → **絶対禁止**（公開Pagesのため）

## 予定・買うものの追加方法（「◯◯を予定に入れといて」と言われたら）

`yotei.json` に追記してコミットする。フォーマット:

```json
{
  "updated": "YYYY-MM-DD",
  "items": [
    {"id": "20260718-toystory", "date": "2026-07-18", "name": "トイストーリー映画いく", "kind": "plan", "place": "イオン幕張"}
  ]
}
```

- `id`: 一意（日付-スラッグ推奨）。既存idは変更しない（端末側の済みチェックがidに紐づくため）
- `date`: `YYYY-MM-DD`。未定なら空文字
- `kind`: `plan`(🎉予定) / `buy`(🛒買いたい) / `reserve`(⏰予約) / `release`(📦発売) / `forgot`(❗買い忘れ)
- `place`: 場所やメモ（任意）
- 削除依頼→該当itemを配列から除く。日付変更→dateを書き換える
- コミット後、GitHub Pagesの再ビルド（約1分）で全端末の「🏠今日」「🛒買うもの」に☁️バッジ付きで反映される

**書き込み手段が無い場合**（GitHubに書けないAI）: 自分で書こうとせず、
「予定の書き込みはMacのClaude Codeに『◯◯を予定に入れといて』と頼んでください」と案内する。
（claude.aiのGitHubコネクタが接続されていれば、このリポジトリの `yotei.json` を直接編集してよい）
